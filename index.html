In graph theory, there is a concept called the
        complete graph. Before we talk about
        what we can do with one, let's look at what a complete graph is.

        A complete graph is a graph in which every vertex is connected to
        ever other vertex in the graph by an edge.
      

      

        Note that, most of the time, there's more edges (lines) than vertices
        (dots). For example:
      


      

        

          This is a complete graph for
          3 vertices:
        


        

        
        

          Move the slider so that there are 2 vertices. Count the edges. Then,
          move the slider so that there are 3 vertices. Count the edges.
        


        

          Going from 2 to 3 vertices increases the number of vertices in the
          graph by 1, but increases the number of edges by
          2.
        

        

          When you add a vertex (a dot), the number of edges (lines) also
          increases. But (except when going from 0 vertices to 1 or from one
          vertex to two)
          the increase in edges is always greater than the increase in
            vertices.

              How many edges (lines) are there in this "complete graph" with 21
              vertices (dots)?
            

            
          

          

            
It's 210! There is an easy way to get this number:

            

              It's
              (one less than the number of vertices)
              times
              (the number of vertices) divided by
              2.
            

          

          

            In this case, that's 20 times 21 divided by 2,
            which equals 210.
          

        

        

          

            

              How many dots are in this equilateral triangle with 20 dots on a
              side?
            

            
          

          

            
It's 210! There is an easy way to get this number:

            

              It's
              (the number of dots on each side of the equilateral
                triangle)
              times
              (the number of dot on each side plus 1)
              divided by 2.
            

            

              In this case, that's 20 times 21 divided by 2,
              which equals 210.
            

          

        

      


      

        Both operations can be rephrased as
        f(n) = n * (n + 1)/2 It's a convenient function! It can
        let you quickly count the edges in much larger complete graphs and get
        the triangular number for far bigger numbers.
